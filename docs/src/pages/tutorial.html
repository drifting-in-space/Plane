1. TUTORIAL

1.1. A simple example of plane running locally.

In this simplified example, we'll essentially be using plane as a souped up inetd. However the concept scales unchanged to multiple drones.

  1. Machines (Virtual or otherwise) that will run jupyter are called 'drones'

1.1.1. Requirements:

  1. Docker
  2. openssl
  3. A local dns server (I use dnsmasq in this tutorial)

1.2. Steps

  1. start nats 

    docker run -d -p 4222:4222 nats:latest --jetstream

    • Configure your local dns server to resolve "nats.home" to 127.0.0.1 For me, this involved adding the following to my /etc/hosts: 

      127.0.0.1	nats.home

  1. start by assigning one controller machine. You will be installing the controller here. Run the following: 

    docker run -p 5354:5354 \
       -v $PWD/controller.toml:/etc/plane/controller.toml \
       plane/controller:latest \
       /etc/plane/controller.toml

    configured with a ./controller.toml file in the same directory: 

    [nats]
hosts = ["nats.home"]

[scheduler]

[dns]
port = 5354
bind_ip = 127.0.0.1
domain_name = plane.home
soa_email = admin@here.localhost
    • configure your local dns server to forward requests for "*.plane.home" to 127.0.0.1:5354

      • for me, this required setting the following lines in dnsmasq.conf 

        server=/*.plane.home/127.0.0.1#5354
  2. provision certs:

    • plane only supports using port 443 at present, this means you will need to provision some self-signed certificates. Here's the openssl command for doing that: 

      openssl req -nodes -x509 -sha256 -newkey rsa:4096 \
        -keyout site-key.pem \
        -out site-cert.pem \
        -days 365 \
        -addext "subjectAltName = DNS:localhost,DNS:*.plane.home" 
mkdir auth
cp *.pem auth

  3. run the drone on the machines 

    docker run \
        -v $PWD/drone.toml:/etc/plane/drone.toml \
        -v /var/run/docker.sock:/var/run/docker.sock \
        -v /$PWD/auth:/etc/plane/auth \
        plane/drone:latest \
        /etc/plane/drone.toml

    configured thusly 

    cluster_domain = "plane.home"

[nats]
hosts = ["nats.home"]

[agent]
#the drones's local ip
ip = "127.0.0.1"

[agent.docker]
network = "plane"

connection = { socket = "/var/run/docker.sock" }

[proxy]
bind_ip = "0.0.0.0"

[cert]
key_path = "/etc/plane/auth/site-key.pem"
cert_path = "/etc/plane/auth/site-cert.pem"
  4. now, spawn a notebook using the plane-cli
docker run -it plane/cli:latest spawn -e JUPYTER_PORT=8080 jupyter/minimal-notebook

and tada! you should get back a backend url linking to a jupyter notebook.
